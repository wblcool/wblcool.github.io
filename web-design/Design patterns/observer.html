    # 初识观察者模式

    概念 : 一个对象(称为subject)维持一系列依赖于它的对象(观察者)，将有关状态的任何变更自动通知给他们.当一个目标需要告诉观察者发生了什么有趣的事情，它会向观察者广播一个通知(可以包括与通知主题相关的特定数据)。当我们不在希望某个特定的观察者获得其注册目标发出的改变通知时，该目标可以将它从观察者列表中删除。
    大概就长这样


    • Subject 维护一系列的观察者，方便添加或删除观察者
    • Observer 为那些在目标状态发生改变时需获得通知的对象提供一个更新接口
    • ConcretreSubject(具体目标) 状态发生改变时,向Observer发出通知,储存ConcreteObserver的状态
    • ConcreteObserver(具体观察者) 存储一个指向ConcreteSubject的引用,实现Observer的更新接口,以使自身状态与目标的状态保持一致。

    《设计模式:可复用面向对象软件的基础》是这样定义的,或许更容易理解:

    一个或多个观察者对目标的状态感兴趣，它们通过将自己依附在目标对象上以便注册所感兴趣的内容,目标状态发生改变并且观察者可能对这些改变感兴趣,就会发送一个通知消息,调用每个观察者的更新方法。当观察者不再对目标状态感兴趣时，它们可以简单的将自己从中分离。

    根据这个可以实现一个简单的Observer模式

    模拟

    根据上述描述先模拟一个目标可能拥有的一系列依赖Observer(不需细看,看名字就知道用途了):

    function ObserverList(){
    this.observerList = []; //存放观察者对象的数组
}

ObserverList.prototype.Add = function(obj){
    return this.observerList.push(obj); 
}

ObserverList.prototype.Empty = function(){
    this.observerList = [];
}

ObserverList.prototype.Count = function(){
    return this.observerList.length;
}
ObserverList.prototype.Get = function (index){
    if(index > -1 && index < this.observerList.length){
        return this.observerList[index];
    }
}

ObserverList.prototype.Insert = function(obj, index){
    var pointer = -1;
    if(index === 0){
        this,observerList.unshift(obj);
        pointer = index;
    }else if(index === this.observerList.length){
        this.observerList.push(obj);
        pointer = index
    }else{
        //...
    }
    return pointer;
}
ObserverList.prototype.IndexOf = function(obj,startIndex){
    var i = startIndex, pointer = -1;
    while (i < this.observerList.length){
        if(this.observerList[i] === obj){
            pointer = i;
        }
        i++;
    }
    return pointer;
}

ObserverList.prototype.RemoveIndexAt() = function(index){
    if(index === 0){
        this.observerList.shift();
    }else if(index === this.observerList.length -1){
        this.observerList.pop();
    }else{
        //...
        }
}
//扩展对象
function extend(obj, extension){
    for(var key in obj){
        extension[key] = obj[key];
    }
}

    这个对象主要用来存放着一系列的观察者对象,并且具有相应添加、删除、插入、获取等一系列的方法。
    下面模拟目标(subject)在观察者列表上添加、删除和通知观察者的能力。













     
     
     
     
     
     








    function Subject(){
    this.observers = new ObserverList();
}

Subject.prototype.AddObserver = function(observer){
    this.observers.Add(observer);
}

Subject.prototype.RemoveObserver = function(observer){
    this.observers.RemoveIndexAt(this.observers.IndexOf(observer, 0));
}

Subject.prototype.Notify = function(context){ //通知observers并访问到concreteObserver实现更新
    var observerCount = this.observers.Count();
    for(var i = 0; i < observerCount;i++){
        this.observers.Get(i).Update(context);
    }
}

//接下来创建一个新的Observer,拥有一个自身的update方法
function Observer(){
    this.Update = function(){
        //...
        }
}

    这样就简单的实现了一个Observer模式,subject只需要调用notify就能通知依赖于它的观察者们实现更新.

    小结

    总结一下就是创建的目标对象(subject)拥有一个属性ObserverList拥有各种管理观察者对象(observer)的方法,subject就可以通过这个调用这个对象上的一系列方法来管理依赖于自己的对象,以及访问到这些观察者从而更新他们的状态。

    下面列举一个实例加深理解。
    我们的目标是向页面中新增多个checkbox,同时有一个checkbox将充当目标,它能够通知其他新增的checkbox需要进行检查并根据它来做出更新。

    详细代码

    先来看看html

    <button id="addNewObserver">Add new Observer checkbox</button>
<input id="mainCheckbox" type="checkbox"/>
<div id="observersContainer"></div>

    这里我们有一个根据用户动态添加checkbox的按钮,一个主要控制的checkbox和一个存放新增checkbox的容器。接下来是js

    var controlCheckbox = document.getElementById("mainCheckbox"),
    addBtn = document.getElementById("addNewObserver"),
    container = document.getElementById("observersContainer");//先获取到这三个DOM元素

//创建一个Subject实例，并扩展到controlCheckbox及目标对象concreteSubject
extend(new Subject(),controlCheckbox);

controlCheckbox["onclick"] = function(){//点击checkbox会发出通知
    controlCheckbox.Notify(controlCheckbox.checked);
}
addBtn.onclick = AddNewObserver;
//实现一个具体观察者concreteObserver
function AddNewObserver(){
   
    var check = document.createElement("input"); //创建一个新的checkbox
    check.type = "checkbox";

   
    extend(new Observer(), check); //将观察者的特性扩展到checkbox上,这里主要就是发起通知时的更新方法update
    
    check.Update = function(value){//重写自定义更新行为
        this.checked = value;
    };
    controlCheckbox.AddObserver(check);//把创建好赋予观察者特性的对象添加进目标(subject)的观察者列表
    
    container.appendChild(check);//将观察者添加到容器上
}

    # 进阶

    Publish/Subscribe(发布/订阅)模式 由于简单的observe模式订阅者和发布者之间有严重的依赖关系,并且也很难传递不同的自定义参数,出现了这种相似设计模式来解决问题。先看一个对比图

    Publish/Subscribe模式使用了一个主题/事件通道,这个通道介于希望收到通知(订阅者)的对象和激活对象(发布者)之间,该事件系统允许代码定义应用程序的特定事件并可以传递自定义参数。与Observer模式不同,它允许任何订阅者执行适当的事件处理程序来注册和接受发布者发出的通知.
    大致的运行模式如下

    var mailCounter = 0; //接受到的消息数量
var subscribe1 = subscribe("inbox/newMessage", function(topic, data){  //订阅1用来呈现消息内容
    console.log("A new message was received: ", topic);
    $(".messageSender").html(data.sender);
    $(".messagePreview").html(data.body);//使用从目标subject传递过来的data,呈现消息
});
var subscribe2 = subscribe("index/newMessage", function(topic, data){//订阅2更新接受消息的数量
    $(".newMessageCounter").html(mailCounter++);
});

publish("index/newMessage" [{
    sender: "hello@google.com",
    body: "Hey there! How are you doing today?"
}]);

//之后通过unsubscribe来取消订阅
unsubscribe(subscribe1);
unsubscribe(subscribe2);

    通过订阅另一个对象的特定任务或活动,当任务/活动发生时获得通知,而不是单个对象直接调用其他对象的方法,这种思想促进了松散耦合。因此产生的好处也是其缺点,此处不做进一步讨论。
    接下来是一个Publish/Subscribe的实现 
    var pubsub = {};
(function(q){
    var topics = {},
        subUid = -1;//之后用于标记订阅
        //发布或广播事件,包含特定的topic名称和参数(比如传递的数据)
        q.publish = function(topic, args){
            if(!topics[topic]){
                return false;
            }

            var subscribers = topics[topic],
                len = subscribers ? subscribers.length : 0;
            while (len--){
                subscribers[len].func(topic, args);
            }

            return this;
        }
        //通过特定的名称和回调函数订阅事件,topic/event触发时执行事件
        q.subscribe = function(topic, func){
            if(!topics[topic]){
                topics[topic] = [];
            }
            var token = (++subUid).toString();
            topics[topic].push({
                token: token,
                func: func
            });
            return token;
        }
        //基于订阅上的标记引用,通过特定topic取消订阅
        q.unsubscribe = function(token){
            for(var m in topics){
                if(topics[m]){
                    for(var i = 0, j = topics[m].length;i < j;i++>){
                        if(topics[m][i].token === token){
                            topics[m].splice(i, 1);
                            return token;
                        }
                    }
                }
            }
            return this;
        }
}(pubsub))

    一个简单的使用案例

    var messageLogger = function(topics, data){ //订阅后处理的回调函数
    console.log("Logging: " + topics + ": " + data);
}
var subscription = pubsub.subscribe("inbox/newMessage", messageLogger);//订阅者监听订阅的topic，一旦topic广播一个通知,订阅者就调用回调函数
//发布者负责发布程序感兴趣的topic,比如：
pubsub.publish("inbox/newMessage", "hello world");
pubsub.publish("inbox/newMessage", ["test", "a", "b", "c"]);
pubsub.publish("inbox/newMessage", {
    sender: "hello@google.com",
    body: "Hey again!"
});
//如果订阅者不想被通知了,也可以取消订阅
pubsub.unsubscribe(subscription);
