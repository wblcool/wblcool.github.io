# 函数防抖

事件频繁触发,但只在事件触发n秒后才执行,如果在事件触发的n秒内又触发了这个事件则以新的时间为准,n秒后执行。

  • 最简单的实现
function debounce(func, wait){
    var timeout;
    return function(){
        clearTimeout(timeout)
        timeout = setTimeout(func, wait);
    }
}
  • 解决this指向及事件对象传递问题
    上面的方法如果传入的函数需要使用事件对象并且引用触发元素则无法正确访问,因此下面作为补充
function debounce(func, wait){
    var timeout;
    return function(){
        var context = this,
            args = arguments;

        clearTimeout(timeout)
        timeout = setTimeout(function(){
            func.apply(context, args)//绑定正确this并提供事件处理函数的事件对象
        }, wait);
    }
}
  • 立即执行
    上面代码比较完善了,但如果我们希望立即执行函数,然后等停止n秒后才可以再次触发,这时可以加个immediate参数,看起来就像这样
function debounce (fn: Function, delay: number, immediate: boolean): Function {
    var timer: number = null
    var debounced: Function = function (): void {
        var context: Function = this
        var arr: number[] = []
        var result: void = null
        window.clearTimeout(timer)
        for (var i: number = 0; i < arguments.length; i++) {
            arr.push(arguments[i])
        }
       // 立即执行一次后不再执行
        if (immediate) {
            immediate = false
            result = fn.apply(context, arr)
        } else {
            timer = window.setTimeout(function () {
                fn.apply(context, arr)
            }, delay) 
        }
        return result //如果有返回值
    }
    //  debounced.cancel = function() { 可以添加一个取消方式,触发取消后不再进行防抖
    //     clearTimeout(timer);
    //     timeout = null;
    // };
    return debounced
}

# 函数节流

持续触发事件，每隔一段时间，只执行一次事件。并且会出现首次触发以及结束后是否执行。
关于节流有两种实现,一是时间戳,一是定时器

# 使用时间戳

触发事件时,保存当前时间戳,减去开始的时间戳(最开始为0),如果大于设置的时间周期,执行函数,然后更新时间戳为当前的时间戳,小于就不执行。

function throttle(func, wait){
    var context, args;
    var previous = 0;
    return function(){
        var now = +new Date();
        context = this,
        args = arguments;
        if(now - previous > wait){
            func.apply(context, args);
            previous = now;
        }
    }
}//事件会在第一次触发立即执行,之后每隔设定的周期触发一次事件,最后停止触发时不会再执行事件

# 使用定时器

触发事件时设置一个 定时器,再触发事件的时候,如果定时器存在就不执行,直到定时器执行然后执行函数。

function throttle(func, wait){
    var timeout;
    var previous;
    return function(){
        context = this;
        args = arguments;
        if(!timeout){
            timeout = setTimeout(function(){
                timeout = null;
                func.apply(context, args)
            }, wait)
            //func.apply(context, args)如果将上面此函数移到此,则会变成立即执行。
        }
    }
}
  1. 第一种事件会立即执行,第二章事件会在n秒后第一次执行
  2. 第一种事件停止触发后没有办法再执行事件,第二种事件停止触发后依然会再执行一次事件 结合一下,如果同时需要开头和结尾都能执行,稍作改动
function throttle(func, wait) {
    var timeout, context, args, result;
    var previous = 0;

    var later = function() {
        previous = +new Date();
        timeout = null;
        func.apply(context, args)
    };

    var throttled = function() {
        var now = +new Date();
        //下次触发 func 剩余的时间
        var remaining = wait - (now - previous);
        context = this;
        args = arguments;
         // 如果没有剩余的时间了或者你改了系统时间
        if (remaining <= 0 || remaining > wait) {
            if (timeout) {
                clearTimeout(timeout);
                timeout = null;
            }
            previous = now;
            func.apply(context, args);
        } else if (!timeout) {
            timeout = setTimeout(later, remaining);
        }
    };
    return throttled;
}

最后我们希望能够控制前后的执行,于是再设置一个参数,根据传值的不同使用不同效果

leading：false 表示禁用第一次执行
trailing: false 表示禁用停止触发的回调

function throttle(func, wait, options) {
    var timeout, context, args, result;
    var previous = 0;
    if (!options) options = {};

    var later = function() {
        previous = options.leading === false ? 0 : new Date().getTime();
        timeout = null;
        func.apply(context, args);
        if (!timeout) context = args = null;
    };

    var throttled = function() {
        var now = new Date().getTime();
        if (options.leading === false) previous = now;
        var remaining = wait - (now - previous);
        context = this;
        args = arguments;
        if (remaining <= 0 || remaining > wait) {
            if (timeout) {
                clearTimeout(timeout);
                timeout = null;
            }
            previous = now;
            func.apply(context, args);
            if (!timeout) context = args = null;
        } else if (!timeout && options.trailing !== false) {
            timeout = setTimeout(later, remaining);
        }
    };
    // throttled.cancel = function() {可取消节流
    //     clearTimeout(timeout);
    //     previous = 0;
    //     timeout = null;
    // }
    return throttled;
}

注意

leading：false 和 trailing: false 不能同时设置。同时设置elseif不会执行,回调中无法调用later,因此无法通过它重置previous的值,导致第一次触发以后,之后再次触发事件时函数会立即执行与设置相反。
