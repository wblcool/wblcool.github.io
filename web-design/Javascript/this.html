# 调用位置

调用位置是函数在代码中被调用的位置，注意不是声明的位置。

# 绑定规则

函数执行的过程中应用四条规则决定this的绑定对象。

# 默认绑定

在通过情况下声明在全局作用域中的变量(var声明)就是全局对象的一个同名属性，此时的this指向全局对象。

function bar(){
    console.log(this.a);
}
var a = 1;
bar(); //1

注意如果使用严格模式，this会被绑定要undefined

function bar(){
"use strict"
    console.log(this);
    console.log(this.a);
}
var a = 1;
bar(); //undefined
       //Uncaught TypeError: Cannot read property 'a' of undefined

# 隐式绑定

根据调用位置是否有上下文对象，或者是否被某个对象拥有或包含，调用位置将this绑定到上下文对象。
不过需要注意一点的是对象属性引用链中只有上一层或者说最后一层在调用位置中起作用。比如:

function bar(){
    console.log(this.a);
}
var obj2 = {
    a: 42,
    bar: bar
};
var obj1 = {
    a: 2,
    obj2: obj2 
};

obj1.obj2.bar(); //42 指向最后一层调用位置

🔶隐式丢失
隐式绑定常见的问题就是丢失绑定对象，从而应用默认绑定把this绑定到全局对象或undefined上(根据是否是严格模式)

function foo(){
    console.log(this.a);
}
function doFoo(fn){
    fn();
}
var obj = {
    a: 2,
    foo: foo
};
var a = "global";
doFoo(obj.foo);//"global"

注意obj.foo实际上是隐式赋值给了参数fn，fn引用的是函数本身，在doFoo中调用时不带任何修饰，因此应用了默认绑定。

# 显式绑定

  • 硬绑定 call、apply、bind

# new 绑定

使用new调用函数，会执行以下操作

  1. 创建一个全新的对象
  2. 这个新对象会被执行[[Prototype]]连接
  3. 这个新对象会被绑定到函数调用的this
  4. 如果函数没有返回其他对象，那么new表达式中的函数调用自动返回这个新对象。

# 优先级

new绑定 》 显式绑定 》 隐式绑定 》 默认绑定

  • 判定顺序

函数如果是new绑定this绑定的是新创建的对象 通过apply、call、bind调用的函数this绑定的是指定对象 根据函数调用的位置，this绑定的是调用位置的上下文对象 以上都不是this使用默认绑定到全局对象，严格模式下绑定到undefined

# 特殊情况

如果在使用call、apply、bind时传入的绑定对象是null或者undefined，这些值在调用时会被忽略，实际应用的是默认绑定规则

function foo(){
    console.log(this.a);
}
var a = 2;
foo.call(null);//2

注意

通常我们会在使用apply传参，或使用bind进行柯里化时传入null，此时this实际绑定到全局对象上，这将会产生一定的风险。
一个更好的解决方案是传入一个特殊空对象Φ = Object.create(null)(又称DMZ对象)，之后this的使用都限制在这个空对象中，不会对全局对象产生任何影响。

🔸另外一个特殊是箭头函数，箭头函数的this绑定的是外层最近一层非箭头函数的this，并且是在定义的位置而不是执行的位置。

# 延伸-软绑定








 








if(!Functio.prototype.softBind){
    Function.prototype.softBind = function(obj){
        var fn = this;
        var curried = [].slice.call(arguments, 1);
        var bound = function(){
            return fn.apply(
                //检查this,如果this被隐式绑定或显式绑定存在则应用绑定的值，如果绑定到全局对象或undefined那么还是绑定之前的指定对象
                (!this || this === (window || global)) ? obj : this,
                curried.concat.apply(curried, arguments)//[...curried,...arguments]
            );
        };
        bound.prototype = Object.create(fn.prototype);
        return bound;
    }
}

不同于bind，绑定后函数的this被指定不能被修改，使用上述软绑定的方式保留了隐式绑定和显式绑定修改this的能力。
