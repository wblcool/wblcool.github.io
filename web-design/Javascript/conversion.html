# 显示类型转换

# toString

基本类型值的字符串化规则 null转"null"，undefined转换为"undefined"，true转换为"true".数字遵循通用规则 ,但极小和极大数字使用指数形式,如:

var a = 1.07*1000*1000*1000*1000*1000*1000*1000
    a.toString()//"1.07e21"

对于普通对象，除非自行定义,否则toString(Object.prototype.toString())返回内部属性[[class]]，如"[object object]".
数组的默认toString方法经过重新定义将所有单元字符串化以后再用“,”连接起来。

let a = [1, 2, 3];
a.toString();//"1,2,3"

# JSON.stringify

JSON.stringify()参数为对象时遇到undefined/function/symbol时会自动将其忽略，数组的话则返回null


 







JSON.stringify(undefined);//undefined
JSON.stringify(function(){})//undefined 
JSON.stringify(
    [1, undeinfed, function(){}, 4]
)//"[1, null, null, 4]"
JSON.stringify(
    {a: 2, b: funtion(){}}
)//"{"a": 2}"

如果对象中定义了toJSON()方法，字符串化时会首先调用该方法，然后用返回值进行序列化。

var b = {
    val: [1, 2, 3],
    toJSON: function(){
        return "[" + this.val.slice(1).join() +  "]";
    }
};
JSON.stringify(b);//""[2,3]""

JSON.stringify()还接受第二、第三个参数,第三个参数指定输出的缩进格式,为正整数时表示每一级缩进的字符数，如果是字符串则用于每一级的缩进。
第二个参数可以是数组或函数，如果是数组则必须是字符串，数组包含序列化要处理的对象的属性名称，不包含的属性则会忽略。比如：

let a={
    b: 42,
    c: "42",
    d: [1,2,3]
}
JSON.stringify(a, ["b", "c"];//"{"b": 42, "c": "42"}")

JSON.stringify(a, function(k, v){
    if(k !== "c") return v;
});// "{ "b": 42, "d":[1,2,3]}"

# toNumber

true =>1 flase=>0 undefined=>NaN null=>0

对象(包括数组)会首先转换为相应的基本类型值，如果返回的是非数字的基本类型值则在按相关规则转换为数字。
为了将值转为基本类型的值，抽象操作toPrimitive会 首先检查是否有valueOf方法，如果有并且返回基本类型值就使用该值进行强制类型转换，没有就使用toString的返回值进行强制类型转换，如果均不返回基本 类型值会产生TypeError错误。

Number("");//0
Number([]);//0
Number(["abc"])//NaN

~x大致等同-(x+1)基于此可以对某些条件判断做一些简化

let a = "hello world";
if(!~a.indexOf("ol")){
    //没有找到匹配
}

有一个坑 [] + {} 和 {} + []返回值分别是"[object object]"和0
第一个{}出现在表达式中会被认为是一个值，即空对象处理,类型转换后为"[object object]"而[]转换后为""
第二个{}出现的 位置会被认为是 一个空代码块不执行任何操作, +[]执行类型转换后的结果是0，如果加了括号则与第一行结果一致

# 宽松相等（隐式类型转换）和严格相等

==允许在相等比较中进行强制类型转换,而===不允许

# 字符串和数字的相等比较

es5规范11.9.3.4-5这样定义
(1)如果Type(x)是数字,Type(y)是字符串,则返回x == ToNumber(y)的结果。
(2)如果Type(x)是字符串,Type(y)是数字,则返回ToNumber(x) == y的结果。

let a = 42;
let b = "42";
a === b ;//false
a == b ;//true

# 其他类型和布尔类型的比较

规范11.9.3.6-7这样说明
(1)如果Type(x)是布尔类型,则返回ToNumber(x) == y的结果。
(2)如果Type(y)是布尔类型,则返回x == ToNumber(y)的结果。

let a = "42"
let b = true;
let c = false;
a == b; //false
a == c;//false

上述例子在转换后类型仍然不同会根据字符串和数字比较的转换规则再进行一次转换。
所以尽量不要使用if(a == true) 改而使用if(a) if(!!a) if(Boolean(a))

# null和undefined比较

null和undefined在==中相等,并且也与自身相等，除此之外其他值 都不和它们相等


let a = null;
let b;
a == b; //true
a  == null;//true
b == null;//true

a == false; //false
b == false; //false
a == ""; //false
b == ""; //false
a == 0; //false
b == 0; //false

！注意：

let a = dosomething();
if(a ==  null){
    //...
}//只有dosomething返回null和undefined时才成立,除此之外的其他值都不成立。包括0,false,""

# 对象和非对象之间的相等比较

(对象和对象之间的比较只需要变量引用地址相同则返回true)

ES5规范11.9.3.8-9如下规定
(1)如果Type(x)是字符串或数字,Type(y)是对象,则返回x == ToPrimitive(y)的结果。
(1)如果Type(x)是对象,Type(y)是字符串或数字,则返回ToPrimitive(x) == y的结果。

let a = 42;
let b = [42];
a == b;//true
[42]首先调用ToPromitive抽象操作返回"42",即"42" == 42然后变成 42 == 42 最后二者相等

注意 
[] == 0;//true
![] == 0 //true
[] == ![] //true (左边"",右边false再转为数字为 0 == 0)
[] == [] //false
{} == !{}//false 左边""[object object] 右边false,左边NaN,右边0
{} == {}//false

最后来看一个有趣的情况 
if（a == 2 && a==3）{
    //...
}

看来好像不太可能,a不会同时等于2和3,但这里不应该说是"同时",因为a==2会在a==3之前执行，因此要实现起来也很简单

let i = 2;
Number.prototype.valueOf = function(){
    return i++;
}
var a = new Number(42);
if(a == 2 && a == 3){
    console.log("this happened");
}//this happened

# 安全运用隐式转换

遵循以下原则可以有效避免出错

  • 如果两边的值中有true或者false，不适用==
  • 如果两边的值中有[]、""或者0，尽量不要使用== 此时最好使用===
