# 模块

模块是自动运行在严格模式下并且没有办法退出运行的javascript代码。在模块顶部创建的变量不会自动被添加到全局共享作用域。 在模块的顶部，this的值是undefined，模块不支持HTML风格的代码注释。

# 导入的语法


import {identifier1, identifier2} from "./example.js";
import * as example from "./example.js";
import {add as sum} from "./example.js"


import sum from "./example" //默认导出的导入不在使用大括号
import sum, {color} from "./example";
import {default as sum, color} from "./example";
//注意在import语句中，默认值必须排在非默认值之前

当从模块中导入一个绑定时，就好像使用const定义的一样，结果是你无法定义另一个同名变量，也无法在import语句前使用标识符或改变绑定的值。 为了最好兼容多个浏览器和node.js环境，要在字符串前包含/、./、或../ 不管在import语句中把一个模块写了多少次，该模块将只执行一次。导入模块的代码执行后，实例化过的模块被保存在内存中，只要另一个import语句引用它就可以重复使用。


import {sum} from "./example.js";
import {multiply} from "./example.js";
import {magicNumber} from "./example.js";

尽管在这个模块中有3个import语句，但example.js将只执行一次，如果同一个应用程序中的其他模块也从example.js导入绑定，那么那些模块与此代码将使用相同的模块实例。

注意：import语句为变量、函数、类创建的只是只读绑定，而不是像正常变量一样简单的引用原始绑定，标识符只有在被导出的模块中可以更改，即便是导入绑定的模块也无法更改绑定的值。 可以执行导入中模块的方法，利用此方法改变导入的变量的值，因为调用此方法的时候会回到导出的模块执行，改动后会自动在导入的变量名上体现。p320

# 加载模块

当type属性的值为module时支持加载模块，可以让浏览器将所有内联代码或包含在src指定的文件中的代码按照模块而非脚本的方式加载。 模块与脚本不同，它是独一无二的，可以通过import关键字指明其依赖的其他文件，并且这些文件必须被加载进该模块才能正确执行。为了支持该功能，type设置为module时，自动应用defer属性。 加载脚本时，defer是可选属性，但加载模块时就是必须属性。一旦遇到type="module"，模块文件便开始下载，直到文档被完全解析模块才会执行。模块按照它们在html文件中的顺序执行。 首先解析模块以识别所有导入语句，每个导入语句都会触发一次获取过程，并在所有导入资源都被加载和执行后才会执行当前模块。 <script type="module>"显式引入和import隐式导入的所有模块都是按需加载并执行的。 1.下载解析module1.js 2.递归下载并解析module1.js导入的资源。 3.解析内联模块。 4.递归下载并解析内联模块中导入的资源。 5.下载并解析module2.js 6.递归下载并解析module2.js中导入的资源。 加载完成后，只有当文档完全被解析后才会执行其他操作。文档执行玩完后会发生一下操作 1.递归执行module1.js中导入的资源 2.执行module1.js 3.递归执行内敛模块中导入的资源。 4.执行内联模块。 5.递归执行module2js中导入的资源 6.执行module2.js

defer脚本会在文档渲染完毕后，DOMContentLoaded事件调用前执行。 浅谈script标签中的async和defer
