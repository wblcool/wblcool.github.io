    # 函数形参的默认值

    参数的默认值 不传或者主动传入undefined时使用,传null会使用null作为其值。

    • 默认参数值对arguments对象的影响 在es5非严格模式下，函数命名参数的变化会体现在arguments对象中，所以参数被赋予新值时，arguments相应的也更新了。在es5严格模式和es6中若使用了默认参数值，arguments对象的行为都将和命名参数保持分离。因此总是可以通过arguments对象将参数恢复为初始值。
    • 默认参数表达式 可以通过函数执行来得到默认参数的值，且可以把前面的参数作为此函数的参数传入。
    • 默认参数的临时死区

    # 处理无命名参数

    • 不定参数 在命名参数前添加3个点表明这是一个不定参数，该参数为一个数组，包含自它之后传入的所有参数。函数length属性统计的是命名参数的数量，不定参数的加入不会影响length的值。pick(object,...key)的length值为1。
    • 无论是否使用不定参数，arguments对象总是包含所有传入函数的参数。

    # 增强的Function构造函数

    构造函数的使用

    var add = new Function("first", "second", "return first + second");
var add = new Function("first", "second = first", "return first + second");
var add = new Function("first", "...args", "return first + args[0]");

    # 展开运算符

    • 展开运算符将数组打散作为各自独立的参数传入函数。
    let values = [25, 50, 75, 100];
Math.max.apply(Math, values);
Math.max(...values);

    使用apply需要手动指定this的绑定。 可以将展开运算符与其他正常传入的参数混合使用，Math.max(...values, 0)表示返回的最小值为0.

    # name属性

    var doAnotherthing = function(){}
var doSomething = function doSomethingElse(){}
var person = {
    get firstName(){
        return "Nicholas"
    },
    sayName: function(){
        console.log(this.name);
    }
}

console.log(doAnotherthing.name);   //"doAnotherthing"
console.log(doSomething.name);   //"doSomethingElse"
console.log(person.sayName.name);  //"sayName"
console.log(person.firstName.name);  //"get firstName"
var doSomething = function(){}
console.log(doSomething.bind().name);  //"bound doSomething"
console.log((new Function()).name);  //"anonymous"

    绑定函数的name属性总是被绑定函数的name属性及字符串前缀"bound"组成。而通过构造函数Function创建的函数名称将是"anonymous"。 注意不要用name属性的值来获取对函数的引用。

    # 判断函数被调用的方法（是否使用new关键字）

    • es5中通过instanceof 判定，即判定this是否是构造函数的实例。(后者原型是否在前者原型链上)
    function Person(name){
    if(this instanceof Person){//instanceof运算符用于测试构造函数的prototype属性是否出现在对象的原型链中的任何位置
        this.name = name;
    }else{
        throw new Error("必须通过new 关键字调用Person")
    }
}

    此方法缺陷在于使用call,apply方法传入Person实例作为第一个参数的话也会成功。

    • 元属性 new.target (元属性指非对象的属性) 当调用函数的[[Construct]]方法时，new.target被赋值为new操作符的目标，通常是新创建对象实例，也就是函数体内的this构造函数，如果调用[[call]]方法，则值为undefined。 还可以通过new.target检查是否被某个特定的构造函数调用

    # 块级函数

    • 严格模式在代码块可以定义函数，会将此函数视作一个块级声明，在if代码块中会被提升至顶部，但一旦if代码块执行结束，doSomething()函数将不在存在。注意使用表达式定义的不会提升。
    • 非严格模式下定义使用function声明的函数会提升至外围函数或者全局作用域的顶部。

    # 箭头函数

    • 箭头函数this,super,arguments及new.target由外层的最近一层非箭头函数决定。
    • 箭头函数没有自己的arguments对象，但始终可以访问外围函数的arguments对象。
    • 箭头函数可以通过call,apply,bind调用,但函数的this值不可变不会受这些方法影响。

    # 尾调用优化

    调用栈是一种数据结构，它基本上记录了我们在程序中的什么位置。如果我们步入一个函数中，我们会把这些数据放在堆栈的顶部。如果我们从一个函数中返回，这些数据将会从栈顶弹出。这就是堆栈的用途。 满足三个条件时尾调用不会创建新的栈帧，而是清除并重用当前栈帧，在递归函数中经常使用。

    1. 尾调用不访问当前栈帧的变量。即不是闭包。
    2. 在函数内部，尾调用是最后一条语句。
    3. 尾调用的结果作为函数值返回。
      满足此三个条件可以被js引擎自动优化。
