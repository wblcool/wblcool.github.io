# video

属性/方法/事件 说明
autoplay(标签) 在chrome浏览器无法自动播放，添加属性muted静音后可自动播放或使用js解决
poster 视频刚加载第一次的封面
src 视频来源地址 可在js中设置
volume 设置返回视频的音量（0-1）可配合input type ="range" 监听oninput事件 使用返回值设置volume的方式控制音量 videoNode.volume = range.value/100
controls videoNode.controls = true;效果同标签上设置是否有控制器
muted videoNode.muted = true;静音
preload 设置或返回是否预加载音频/视频
duration 返回视频总长度 单位：秒
currentTime 返回当前视频播放长度 可提前设置则载入时即到设定位置
  • 播放一个视频暂停或退出时保存此数据在本地第二次加载时设置此数据即可继续播放
  • 快进、后退，通过绑定事件设置此属性值变化达到快进后退效果 (单位：秒)
foward.onclick = function(){
videoNode.currentTime += 30;//每次电击快进30s
}
networkState 返回当前视频的网络状态 返回值0-3
  • 0 = NETWORK_EMPTY - 音频/视频尚未初始化
  • 1 = NETWORK_IDLE - 音频/视频是活动的且已选取资源，但并未使用网络
  • 2 = NETWORK_LOADING - 浏览器正在下载数据
  • 3 = NETWORK_NO_SOURCE - 未找到音频/视频来源
currentSrc 返回视频/音频的src（加载能播放的情况下才能正确返回否则为空字符串） 只读
paused 检查视频是否已经暂停 布尔值
played 返回一个TimeRanges对象 该对象表示用户已经播放或看到的视频范围其具有以下属性
  • length - 获得音频/视频中已播范围的数量
  • start(index) - 获得某个已播范围的开始位置
  • end(index) - 获得某个已播范围的结束位置
注意：首段已播范围的下标是 0不是1。
loop 设置或返回视频播放完毕后循环播放
playbackRate 查看/设置视频播放速度 -0.5向后半速，-1.0向后正常速度,1.0一倍速,2.0两倍速
readyState 表示音频/视频元素的就绪状态(类似ajax)
  • 0 = HAVE_NOTHING - 没有关于音频/视频是否就绪的信息
  • 1 = HAVE_METADATA - 关于音频/视频就绪的元数据
  • 2 = HAVE_CURRENT_DATA - 关于当前播放位置的数据是可用的，但没有足够的数据来播放下一帧/毫秒
  • 3 = HAVE_FUTURE_DATA - 当前及至少下一帧的数据是可用的
  • 4 = HAVE_ENOUGH_DATA - 可用数据足以开始播放
play() 视屏自动播放，chrome无法自动播放，会报错，可通过绑定事件再模拟事件发生使其自动播放。
window.onclick = function(){
videoNode.play();
}
pause() 暂停播放
load() 重新加载音频/视频元素
用于在更改来源或其他设置后对音频/视频元素进行更新
requestFullscreen() 用于发出异步请求使元素进入全屏模式 注意兼容写法（注意s大小写） 使用语法
ended 属性: 返回布尔值 视频是否播放结束
事件：视频播放结束时触发
timeupdate 音/视频播放位置改变时触发
播放和移动播放位置时刻被调用
seeked seeked 事件在用户已移动/跳跃到音频/视频（audio/video）中的新位置时触发。
seeking 属性(boolean)：返回用户是否在音频/视频中寻址
事件：在用户开始移动/跳跃到新的音频/视频（audio/video）播放位置时触发。从单词就能看出上面二者区别
volumechange 当音量更改时触发
加载过程中依次触发事件
  • loadstart:加载过程开始
  • durationchange:指定音频/视频的时长数据发生变化时(音频/视频加载后，时长将由 "NaN" 变为音频/视频的实际时长)
  • loadedmetadata:指定的音频/视频的元数据已加载时(时长、尺寸、文本轨道)
  • loadeddata:当前帧的数据已加载，但没有足够的数据来播放指定音频/视频的下一帧时触发
  • progress:浏览器正在下载指定的音频/视频时，会发生 progress 事件
  • canplay:浏览器能够开始播放指定的音频/视频时
  • canplaythrough:浏览器预计能够在不停下来进行缓冲的情况下持续播放指定的音频/视频时

注意上面事件只在视频开始加载过程触发
