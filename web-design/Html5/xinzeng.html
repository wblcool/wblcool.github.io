# 新增标签

# 结构标签

  • <article> 标记一篇文章
  • <header> 标记定义一个页面或一个区域的头部
  • <nav> 标记定义导航链接
  • <section> 标记定义一个区域
  • <aside> 标记定义页面内容部分的侧边栏
  • <hgroup> 标记定义文件中一个区块的相关信息
  • <figure> 标记定义一组媒体内容以及他们的标题
  • <figcaption> 标记定义figure元素的标题
  • <footer> 标记定义一个页面或一个区域的底部
  • <dialog> 标记定义一个对话框(会话框)类似微信

# 多媒体标签

  • <audio> 标记定义一个音频
  • <video> 标记定义一个视频
  • <source> 标记定义媒体资源 详细
  • <canvas> 标记定义一个视频 详细
  • <embed> 标记定义外部可交互内容或插件，可引入flash、svg等

# 状态标签

  • <meter> 状态标签(实时状态显示: 气压、气温)
    <meter value="200" min="20" max="380" low="200" high="240" optimum="220"></meter>

    <meter value="0.2" low="0.5"></meter>
  • <progress> 状态标签(任务过程: 安装、加载)
    <progress value="50" max="100"></progress>

    <progress max="100"></progress>

# 列表标签

  • <datallist> 为input标记定义一个下拉列表,配合option
<input placeholder="请选择您喜欢的水果" list="fruitList" />

    <option value="apple">apple</option>
    <option value="banana">banana</option>
    <option value="pear">pear</option>
    <option value="pineapple">pineapple</option>
    <option value="strawberry">strawberry;</option>
</datalist>


  • <details> 标记定义一个元素的详细内容,配合summary
<details> <!--open="open"是否默认打开 -->

# 注释标签

  • <ruby> 标记定义注释或音标
  • <rt> 标记定义对ruby的注释内容文本
  • <rp> 告诉那些不支持ruby元素的浏览器如何去显示

<p>我们来<ruby>夼<rp>(</rp><rt>Kuang</rt><rp>)</rp></ruby>一个话题</p>

我们来夼(Kuang)一个话题

  • <mark> 标记定义文本(黄色选中)
  • <output> 标记定义一些输出类型,计算表单结果配合oninput事件

   =<output name="x" for="a b"></output> <!-- for:定义输出域相关的一个或多个元素 name:定义对象唯一名称-->
</form>

# 新增属性

# input type

email url tel number date month week time datetime datetime-local range search color(颜色选择板) range的美化

# 表单属性

  • <autocomplete="on/off"> <input type="text" name="username" autocomplete/> 是否自动补全
  • <autofocus> <input type="text" name="username" autofocus/> 自动聚焦
  • <multiple> <input type="file" name="file" multiple/> 文件可多个上传
  • <required> <input type="text" name="username" required/>表单验证

# 链接属性

  • sizes<link rel="icon" href="icon.gif" type="image/gif" sizes="16x16"> 指定图标尺寸
  • media <a media="value">规定目标 URL 是为什么类型的媒介/设备进行优化的
  • type <a type="value">规定目标urlMIME类型
  • ref <a rel="value"> 不是新属性,rel 属性规定当前文档与被链接文档之间的关系。注意下ref=""noreferrer

# 其他属性

  • defer
  • async

二者都是异步加载，都可用于避免阻塞，区别在于defer需要等待DOM结构加载完成即触发DomContentLoaded事件时才会执行，并且如有多个defer会按照先后顺序依次执行。 async没有要求，只要加载完成即刻执行，与顺序无关
